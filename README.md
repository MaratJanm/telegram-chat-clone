# Telegram Chat Clone

Простой клон чата Telegram для одного пользователя.
Стек: React + TypeScript + Tailwind CSS + Vite (фронтенд), FastAPI (бэкенд), Docker + Docker Compose.

## Структура проекта
Понял проблему! Ты имеешь в виду, что при использовании моего первого варианта (с обычными отступами и символами ├──) на GitHub дерево "уходит влево" — ветки ломаются, всё съезжает, и выглядит криво.
Это классическая проблема: GitHub Markdown не поддерживает произвольные unicode-символы дерева (├──, │, └─) в обычных списках надёжно. На десктопе может выглядеть нормально, а на мобильном или в некоторых браузерах — полностью ломается и "уходит влево".
Лучшее решение для GitHub — чистый код-блок с моноспейсным шрифтом
Оберни дерево в тройные обратные кавычки (```) — это гарантированно сохранит все отступы и символы одинаково на всех устройствах.
Вот рабочий вариант с короткими комментариями (строки ≤ 80 символов, чтобы точно без горизонтального скролла даже на телефоне):

telegram-chat/
├── backend/
│   ├── main.py          # Основной бэкенд с API
│   ├── Dockerfile       # Docker-образ бэкенда
│   └── requirements.txt # Зависимости Python
├── frontend/
│   ├── src/
│   │   ├── App.tsx         # Главный компонент React
│   │   ├── index.css       # Основные стили
│   │   ├── main.tsx        # Точка входа React
│   │   ├── vite-env.d.ts   # Типы для Vite
│   │   ├── api/
│   │   │   └── messages.ts # API-клиент сообщений
│   │   ├── components/
│   │   │   ├── Chat.tsx         # Основной чат
│   │   │   ├── ChatHeader.tsx   # Заголовок чата
│   │   │   ├── Message.tsx      # Одно сообщение
│   │   │   ├── MessageInput.tsx # Ввод сообщения
│   │   │   └── MessageList.tsx  # Список сообщений
│   │   └── types/
│   │       └── index.ts    # Типы TypeScript
│   ├── public/
│   ├── Dockerfile          # Docker-образ фронтенда
│   └── nginx.conf          # Nginx-конфиг для продакшена
├── docker-compose.yml      # Запуск контейнеров
├── .env                    # Переменные окружения
└── README.md               # Описание проекта


## Запуск локально (рекомендуется)

```bash
docker compose up --build
```

Фронтенд: http://localhost:3000
API: http://localhost:8000

## Запуск без Docker
Бэкенд:
```bash
cd backend
pip install -r requirements.txt
uvicorn main:app --reload
```
Фронтенд:
```bash
cd ..
cd frontend
npm install
npm run dev
```

## API

| Метод   | Эндпоинт        | Описание                   |
|---------|-----------------|----------------------------|
| GET     | `/api/messages` | Получить все сообщения     |
| POST    | `/api/messages` | Отправить новое сообщение  |
| DELETE  | `/api/messages` | Очистить все сообщения     |
